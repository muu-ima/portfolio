# tools-hub

## Overview

tools-hub は、海外販売向けの利益計算、送料シミュレーション、為替表示、発送情報管理をまとめた業務支援ハブです。

単純な計算機ではなく、日々の販売判断で迷いやすい為替、VAT、カテゴリ手数料、Payoneer 手数料、配送重量、実送料をまとめて扱えるようにしています。UK / US 向けの販売条件を分けつつ、画面上ではできるだけ迷わず入力できる形に落とし込んでいます。

## Main Features

- UK 向け利益計算
- US 向け利益計算
- 通常計算
- 逆算モード
- 最安値 / Platform モード
- GBP / USD / JPY の為替換算
- UK VAT ルールの判定
- 135 GBP threshold rule の考慮
- VAT 込み / VAT 抜き価格の扱い
- カテゴリ手数料、Payoneer 手数料、為替手数料の計算
- 実重量 / 容積重量による送料シミュレーション
- 小型包装物、FedEx、EMS の配送候補比較
- WordPress Products CPT と連携した発送情報管理
- SKU、カテゴリ、重量、サイズ、実送料の検索・編集
- 認証付きの発送情報登録フォーム

## UK Profit Calculation

UK 版では、JPY ベースの仕入れ値と送料を持ちながら、GBP の販売価格、VAT、カテゴリ手数料、Payoneer 手数料、為替手数料を組み合わせて最終利益を計算します。

特に難しいのは、画面上の販売価格が VAT 込みなのか、VAT 抜きなのかによって計算の前提が変わる点です。`135 GBP` の threshold rule も絡むため、単純に 20% を足すだけでは判断できません。

実装では、GBP を基準にした計算ロジックへ正規化し、VAT 判定、手数料、還付、為替手数料、最終利益率まで同じ流れで扱っています。

## USD To GBP VAT Flow

tools-hub の特徴的な部分は、USD 売値をそのまま円換算するだけではなく、一度 GBP に変換して UK VAT ルールへ通している点です。

US ドル販売価格を扱う場合でも、VAT 判定や 135 GBP ルールは GBP ベースで見る必要があります。そのため、USD → GBP のクロスレートを作り、GBP 側の計算エンジンに通したあと、表示用に USD / JPY の値へ戻しています。

この流れによって、USD で入力していても、UK 側の VAT 条件を外さずに利益判断できるようにしています。

## Reverse Calculation

逆算モードでは、目標利益率から必要な販売価格を求めます。

通常計算と別ロジックで雑に逆算するのではなく、順行計算の関数を使いながら二分探索で価格を探しています。そのため、VAT 判定、送料、手数料、為替手数料、還付金の扱いが通常計算とずれにくい構成です。

## Shipping Manager

Shipping Manager は、商品の重量、サイズ、実送料を管理するための画面です。

WordPress Products CPT と REST API で連携し、Next.js 側では SKU、カテゴリ、重量、サイズ、実送料を検索・編集できる UI を用意しています。

送料計算では、実重量だけでなく容積重量も扱います。FedEx などで使う容積重量を計算し、実重量と比較して大きいほうを配送適用重量として扱います。小型包装物は重量、長さ、三辺合計の条件も見るため、配送方法ごとの制約を別々に判定しています。

## Technical Challenges

- 日本円ベースの仕入れ値を、GBP / USD の販売価格とつなげること
- USD 売値を一度 GBP に変換し、UK VAT ルールに通すこと
- VAT 込み / VAT 抜きの価格を、計算モードごとに正規化すること
- 135 GBP threshold rule を利益計算全体に反映すること
- カテゴリ手数料、Payoneer 手数料、為替手数料、税還付をまとめて扱うこと
- 通常計算と逆算モードで、計算結果の前提がずれないようにすること
- 実重量、容積重量、配送方法ごとの制約を送料判断に反映すること
- 複雑な計算条件を、画面上ではシンプルに扱えるようにすること
- WordPress 商品データと Next.js の管理画面を REST API でつなぐこと

## Good Points

- 複雑な越境ECの判断材料を、ひとつの計算画面にまとめられている
- JPY、GBP、USD をまたぐ計算を、販売判断に使える形まで落とし込めている
- UK VAT や 135 GBP ルールなど、実務で見落としやすい条件をロジック化している
- 通常計算と逆算モードが同じ計算前提を共有していて、結果のズレが起きにくい
- 送料計算を実重量だけでなく容積重量や配送方法の制約まで含めて扱っている
- WordPress Products CPT とつなぎ、単発の計算機から発送情報管理へ広げられている
- 画面は軽く見えても、裏側にはかなり実務寄りの計算モデルが入っている

## Notes From Code Review

コードを見た印象として、tools-hub は「入力フォーム付きの計算機」よりも、越境ECの販売判断を支える小さな業務システムに近いです。

`lib/profitCalc.ts` に UK 計算の中心があり、VAT 判定、GBP / USD / JPY の変換、還付金、逆算までが集約されています。`calculateFinalProfitDetailFromUSD` では、USD を GBP に変換して既存の GBP ロジックに通しており、販売通貨と税制判定通貨がずれる問題を吸収しています。

Shipping Manager 側は、WordPress 商品メタを Next.js 側で扱いやすくするための業務画面です。商品データ、重量計算、送料候補、登録フォームを分けているため、利益計算ツールから発送情報管理へ拡張していける土台があります。
